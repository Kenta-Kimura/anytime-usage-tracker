# エニタイム訪問履歴・店舗移籍リスク管理

エニタイムフィットネスの店舗利用履歴をローカルで管理し、公開PDFの移籍ルールに近づいている店舗を確認するための個人用Webアプリです。

移籍リスク判定とは別に、月会費をどれだけ活用できているかを月単位で確認する費用分析機能も備えています。

## 使い方

1. `index.html` をブラウザで開く、またはこのフォルダでローカルサーバーを起動します。
2. 所属店舗を設定します。
3. 利用日、利用店舗、メモを登録します。
4. 30日集計、リスク判定、費用分析、履歴、カレンダーを確認します。

データはブラウザのローカルストレージに保存され、外部には送信されません。

ローカルサーバーで開く例:

```sh
python3 -m http.server 4174 --bind 127.0.0.1
```

起動後、`http://127.0.0.1:4174/` を開きます。

## 主な機能

- 所属店舗の設定
- 利用日、利用店舗、メモの手入力登録
- 過去に入力した店舗名の候補表示
- 利用履歴の一覧、編集、削除
- 直近30日ごとの店舗別利用回数と割合表示
- 移籍条件に近づいている店舗の警告表示
- 今日または予定日に特定店舗を利用した場合のリスク判定
- あと何回利用すると危険かの目安表示
- カレンダー形式での履歴と危険日の表示
- カレンダー上の危険日にマウスオーバーした際の対象店舗表示
- 月会費ベースの利用単価可視化

## 画面構成

- `概要`: 所属店舗、移籍ルールの目安、直近30日集計、警告店舗を表示します。
- `記録・判定`: 利用履歴の登録、今日の利用判定、行動シミュレーターを表示します。
- `費用分析`: 今月の利用単価、月間目標、追加利用時の単価、過去月推移を表示します。
- `履歴・カレンダー`: カレンダーと利用履歴を同じ画面に表示します。履歴の編集もこの画面内で完結します。

## 移籍リスク判定の前提

参照元: http://af-jp.net/pdf/移籍ルール.pdf

- 過去30日間に4回以上利用がある
- そのうちホームクラブ以外の同一クラブの利用が51%以上ある
- 上記が直近30日とその前30日の両方で続く場合に移籍リスクが高まる
- 30日間に1回以上の利用があり、そのすべてがホーム以外の同一クラブである状態が3期間続く場合も注意対象

このアプリでは、直近30日単体で条件に近い、または到達した場合は「要注意」、直近30日とその前30日の両方で同じ非ホーム店舗が条件に到達した場合は「危険」として表示します。個人の行動判断を補助するためのもので、公式な判定ではありません。

現在の自動判定は上記の連続条件を中心にしたルール1対応です。3期間連続のルール2は未実装で、画面上にもその旨を表示しています。

## 費用分析の前提

- ホーム店舗「コトニアガーデン新川崎」の月会費として、月額 `7,678円` を固定値で扱います。
- 費用分析は移籍リスク分析とは独立しています。
- 移籍リスク分析は直近30日のローリング集計、費用分析は月単位の集計です。
- `月会費 ÷ その月の利用回数` を「1回あたりの実質費用」として表示します。
- 利用回数が0回の月は、1回あたり費用を表示せず `未利用` と表示します。
- 月間目標利用回数はローカルストレージに保存されます。
